nginx root和alias区别

发表于 2018-10-16   |   分类于 nginx

1 区别

root与alias主要区别在于nginx如何解释location后面的uri，这会使两者分别以不同的方式将请求映射到服务器文件上。
root的处理结果是：root路径＋location路径
alias的处理结果是：使用alias路径替换location路径

2 root例子

location ^~ /t/ {
root /www/root/html/;
}
如果一个请求的URI是/t/a.html时，web服务器将会返回服务器上的/www/root/html/t/a.html的文件

3 alias例子

location ^~ /t/ {
alias /www/root/html/new_t/;
}

如果一个请求的URI是/t/a.html时，web服务器将会返回服务器上的/www/root/html/new_t/a.html的文件。注意这里是new_t，因为alias会把location后面配置的路径丢弃掉，把当前匹配到的目录指向到指定的目录

VNC

发表于 2018-10-12   |   分类于 concept

1. VNC是什么？

VNC（Virtual Network Computing），为一种使用RFB协议的显示屏画面分享及远程操作软件。此软件借由网上，可发送键盘与鼠标的动作及即时的显示屏画面。

这个定义很简洁也很精确，有几个关键词，一个是RFB协议，另外就是显示屏画面分享和远程操作。所以如果我想实现这么一个软件，我要怎么做呢？显然这个是C-S架构，在显示屏分享这个方向Server要做的就是不断做屏幕截图，然后发送到客户端，在远程操作的这个方向Client就是要把用户的鼠标、键盘操作编码发送到服务端，然后服务器负责解析执行。这里面的主要技术挑战就是如何编码，减少带宽。

2. VNC 流程图

  1. 用户从浏览器点击登陆
  2. 前端发起一个API AllocateVMVNCSession的请求
  3. API层收到请求，调用access的 AllocateVMVNCSession
  4. access向数据库插入一条记录,返回一个session-id
  5. 前端收到这个请求后就会带着session-id跳转到如下的这个新页面 https://console.dev.ucloudstack.com/vncproxy/vnc.html?token=vncssid-siI-uY1ig, 这个页面是个前端项目，使用开源项目 no vnc
  6. no vnc根据读到的token发起websocket请求如下 wss://console.dev.ucloudstack.com/websockify?token=vncssid-siI-uY1ig
  7. nginx接收到这个请求会匹配到location /websockify这个字段，进而转发到wsbackend, 这个wsbackend起的是cmpbe gateway/vncproxy这个项目

  1. vncproxy的session manager会根据收到的session-id来获取VNC host的ip，port及password, 这一步是通过GetVMVNCSession这个函数获取到的，它的步骤是先通过session-id换算出对应的vm-id，然后查vm表得到对应的ip等信息，然后转发到huanghe的vnc-tcp-proxy

  2. huanghe的vnc-tcp-proxy读取传过来的ip，port然后再转发到后端的宿主机上的端口

  3. 宿主机根据端口转发到上面真正的虚拟机

快速排序

发表于 2018-10-10   |   分类于 alogrithm

快速排序的思路是先找一个pivot，也就是基准值，然后把整个数组分成三部分，pivot的左边部分是一个所有小于pivot值组成的数组，然后是pivot，然后右边的部分是大于pivot值组成的数组，假设左边和右边的数组是已经排序好的，那只要把这三个数组连起来就行，然后就开始分别递归左边和右边的数组，递归的基准条件是数组只有0个或1个元素，至于为什么有0个元素的情况，试想下当数组是两个元素的时候，做一个pivot的左右分组必然会有一个是0个元素。

下面是我的golang实现，这里在作pivot的时候用了循环

```
package main

import (
“fmt”
“math/rand”
)

// 数组长度
const (
Length = 10
)

func main() {
arr := make([]int, Length)
for i := 0; i < Length; i++ {
arr[i] = rand.Intn(Length)
}

fmt.Println("original", arr)

fmt.Println("after", quickSort(arr))

}

func quickSort(a []int) []int {
if len(a) <= 1 {
return a
}

pivot := a[0]
less := make([]int, 0)
great := make([]int, 0)
for i := 1; i < len(a); i++ {
    if a[i] <= pivot {
        less = append(less, a[i])
    } else {
        great = append(great, a[i])
    }
}
return append(append(quickSort(less), pivot), quickSort(great)...)

}

再一次认识大O

发表于 2018-10-09   |   分类于 alogrithm

在学习算法的过程中不可避免地会多次遇到大O表示法，如果不理解大O表示法那么之后的学习会非常痛苦，大O表示法表示的到底是什么呢？是算法的运行时间吗？

这个回答不准确，大O表示法确实是想度量算法的时间维度，但它是个关于样本集n的函数，它是想表示随着样本集的增大，整个算法的时间是如何增长的。举个例子，
比如要在一个整型数组中查找一个特定的整数，在10个整数中找假如是10秒，那么在100个整数中找是多少秒呢？是100秒吗？在10000个整数呢？如果答案是10000秒，
那么我们可以说这个时间复杂度是线性的，但如果是用二分查找分，它是O(log n)的，因此100个整数只需要7秒左右，它不是线性的，比线性的要快的多。

所以总结下，大O表示法是用来表示算法的时间与样本集个数的函数，随着样本集变大，算法时间是怎么来变的这么一个函数。

go的字符串处理

发表于 2018-09-06   |   分类于 go

rune

说到golang的字符传首先得说下golang的一个专有类型rune，就我使用过的几门语言还没有遇到过这个rune，它实际上是一个int32类型，但为什么还要单独定义一个rune类型呢? 这里先说下结论: 它是用来表示一个Unicode code point。Unicode是对于ASCII的扩展(http://www.unicode.org/)，它收集了这个世界上所有的符号系统，包括重音符号和其它变音符号，制表符和回车符，还有很多神秘的符号，每个符号都分配一个唯一的Unicode code point。在第八版本的Unicode标准收集了超过120,000个字符，涵盖超过100多种语言。这些在计算机程序和数据中是如何体现的呢？通用的表示一个Unicode码点的数据类型是int32，在golang里就用rune表示。

UTF-8

golang默认就是支持unicode的，所以fmt.Println(“你好，中国”) 是没有问题的，但这些Unicode字符是怎么存储成文件的呢？比如go的源码文件是怎么存的呢？一个简单的方式就是用[]int32的形式，也就是[]rune,每一个int32存一个Unicode code point，但是这样太占空间了，很多常用的英文字符根本就用不到4个字节，可想而知如果使用这个方式存储会浪费很大的存储空间。那golang是怎么存储的呢？golang是使用了一种UTF-8的编码技术, 我想可能是因为UTF-8的发明人Ken Thompson和Rob Pike本身就是golang的发明人吧，哈哈。那UTF-8是一种怎样的编码技术呢？

0xxxxxxx runes 0-127 (ASCII)
110xxxxx 10xxxxxx 128-2047 (values <128 unused)
1110xxxx 10xxxxxx 10xxxxxx 2048-65535 (values <2048 unused)
11110xxx 10xxxxxx 10xxxxxx 10xxxxxx 65536-0x10ffff (other values unused)

它采用变长的方式来编码，可以看到ASCII被编成了只有一个字节，汉字很多是三个字节。这样一来空间是被大大减小了，但一个问题是在程序里用UTF-8处理字符串不是
