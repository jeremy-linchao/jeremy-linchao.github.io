Git Branch 12.1

https://git-scm.com/book/en/v2/Git-Branching-Branches-in-a-Nutshell

每一个commit对象包含了几样东西，1）size 2）一个指向tree的指针 3）一个指向parent的指针 4）author和committer

所谓的分支就是一个可移动的指针，指向某个commit对象，创建一个分支git branch就是创建了这么一个指针，这个命令不会移动HEAD指针

那Git如何知道现在是工作在哪个分支呢？通过HEAD指针

可以用Git checkout 移动HEAD指针来指向不同的分支

Git 本地操作 11.30

查看提交历史

git log -p Show the patch introduced with each commit. 加上-p可以看到每次提交的修改之处

git log -p - - stat Show statistics for files modified in each commit.

git log - -pretty=format 可以以指定格式输出

Undo

git commit - - amend 第一次提交忘记用git add把相关文件加进去了，可以加完之后用这个命令，这样就避免做了两次提交

git reset HEAD filename 一下子把多个文件加到staged区域里去了，但实际上想做的是分为多次提交，那可以用这个命令把相应的文件拿出来

git checkout - - file 修改了某个文件但还没有加到staged区域，可以使用这个命令把那些修改重置

Remote

git remote -v 查看远端server的url，origin是指默认的server git当时clone的

git remote add 配置远端server，如果是clone的这步不需要，因为会把origin当做默认的远端server

git fetch [remote-name] 这个命令会从远端server拿自己没有的数据，如果是clone的用git fetch origin就可以，注意这个命令只是下载，不会merge，同时也不会修改本地的工作目录

Angular 11.28

调试获取scope元素

右键审查元素，然后angular.element($0).scope()

Angular的事件机制

  1. emit机制： The event life cycle starts at the scope on which $emit was called. All listeners listening for name event on this scope get notified. Afterwards, the event traverses upwards toward the root scope and calls all registered listeners along the way. 意思是emit首先会被当前scope下的listeners获取到，之后会向上级listeners传
  2. broadcast机制：The event life cycle starts at the scope on which $broadcast was called. All listeners listening for name event on this scope get notified. Afterwards, the event propagates to all direct and indirect scopes of the current scope and calls all registered listeners along the way. 意思是borodcast会被当前scope下的listener获取到，之后会向下级的listeners传递

Angular 11.22

When a Controller is attached to the DOM via the ng-controller directive, Angular will instantiate a new Controller object, using the specified Controller’s constructor function. A new child scope will be created and made available as an injectable parameter to the Controller’s constructor function as $scope.

这段话翻译过来的意思是说当ng-controller被加到一个dom元素中会发生什么？首先Angular会来初始化一个controller对象，用的就是controller这个构造函数，然后会有一个新的子scope会被创建出来，被当成一个参数传到构造函数里去

Use controllers to:

  • Set up the initial state of the $scope object.

  • Add behavior to the $scope object.

    使用controller主要就来做两件事，第一件是初始化$scope这个变量，另一件事是给$scope加上行为

为什么要打包

浏览器在解析js的时候是按顺序来作解析的，如果你把基础依赖放到最后，那浏览器在解析到前面几行就会报错，所以一般要把依赖最大的放到最上面，但是这样比较麻烦，所以打包工具的的用途就体现出来了，他能帮助我们解决打包的问题
