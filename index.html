关于异步

为什么需要异步？

  1. 前端无疑是异步用的最多的地方，举个例子，浏览器中javascript的执行与UI渲染是在同一个线程，这样的话如果javascript如果同步去获取远端一个资源，在这个期间UI的渲染和响应就是停止的，用户点击任何东西是没有反应的，用户体验非常差，通过异步的方式可以消除这个阻塞现象

  2. 在内核IO调用只有阻塞与非阻塞，在调用阻塞IO时，应用程序需要等待I/O完成才返回结果；而非阻塞I/O则不带数据直接返回，要获取数据，还需要通过文件描述符再次读取

  3. 前面提到了为了获取完整的数据，应用程序需要重复调用I/O操作来确实是否完成，这种重复调用判断操作是否完成的技术叫做轮询，有以下几种形式：

    a) read b)select c)epoll 4) 这些个技术对于应用程序而言，仍然只能算是一种同步，因为应用程序需要等待I/O完全返回，等待期间，CPU要么用于遍历文件描述符的状态，要么用于休眠等待时间发生

  4. Linux下面存在一种异步IO方式叫AIO，但这个只在linux下有，并且仅支持O_Direct方式读取

  5. 现实场景下的异步IO： 线程池模拟异步I/O

    一个线程进行计算处理，让其他线程进行非阻塞I/O加轮询获取数据，通过线程间通信将I/O的数据进行传递

  6. 我们说node是单线程的，这里的单线程仅仅只是javascript执行在单线程中，内部完成I/O任务的另有线程池

ES6 Generate 函数

  1. generator函数可以用来做一种异步编程解决方案

  2. generator函数有多种理解角度，从语法上generator函数是一个状态机，封装了多个内部状态。它返回一个遍历器对象，所以generator函数除了是个状态机，还是一个遍历器对象生成函数

  3. generator函数是一个普通函数，但有两个特征，1）function关键字与函数名之间有一个星号 2)函数体内部使用yield语句，定义不同的内部状态

  4. 调用Generator函数，返回一个遍历器对象，代表Generator函数的内部指针。以后，每次调用遍历器对象next方法，就会返回一个有着value和done两个属性的对象。value属性表示当前的内部状态的值，是yield语句后面那个表达式的值；done属性是一个布尔值，表示是否遍历结束

  5. yield句本身没有返回值，或者说总是返回undefined。next方法可以带一个参数，该参数就会被当作上一个yield语句的返回值

Node 工程化

  1. 目录结构：模块应用遵循CommanJS，web应用通过框架或者自行按MV*组织

  2. 构建工具：make(需要bash编程)，Grunt(本身用nodejs写)

  3. 编码工具：JSLint和JSHint，一旦团队约定了编码规范的详细细则，则可以生成一份规则文件，扫描工具或者编辑器能够通过该规则文件对源码进行扫描

  4. 代码审查：1）功能点是否正确完成 2）编码风格是否符合规范 3）单元测试是否有同步添加

  5. 部署环境：代码 -> stage(测试环境) -> 预发布 -> 生产，之所以有个预发布环境是因为测试环境只能满足功能的正确性，而预发布环境可以做数据相关的检查，它的数据较为接近线上真实的数据

  6. 部署操作：nohup node app.js & ，停止和重启是通过相应的pid写进app.id文件下

  7. 日志记录：通常的API，尽量不要隐藏错误，不要通过try/catc捕获异常，事实上，日志通常是服务于业务的，尽量由最上层的调用者捕获记录，底层调用或中间层调用出现的异常只要正常传递给上层的调用方即可

对于异步编程的理解

1）为啥需要异步编程的解决方案？

​ 有一个基础知识是需要理解的，那就是CPU的运行速度是比外部IO的运行速度快的多很多，比如读一个文件的程序，CPU在执行读文件的时候必须等待文件读取到才能继续往下执行，那这段时间的CPU就被闲置了，并且也没有响应，传统的做法就是利用多线程，多线程会引入锁、同步等问题，而node别出溪径，走的是异步的方式

2） 异步方案的难点

  1. try catch无法捕捉异常，因为try catch本身是捕捉当次执行时的报错，而异步的方式那个程序在那个刻还没执行，自然也就catch不到，node在异常处理上形成一种约定，将异常作为回调函数的第一个参数传回，如果为空，则表明没有异常抛出，如果自己编写异步方法必须遵循如下原则：

    必须执行调用者传入的回调函数

    正确传递回异常供调用者判断

  2. 函数嵌套过深：多个操作存在依赖关系，比如遍历目录，查询数据库做渲染​

Javascript 原型prototype

为什么要有原型

考虑一个需要定义一个person对象

这个person会有名字、年龄和工作的属性，这些都是每个person对象不一样的地方，但是如果我们要定义一个sayName方法，由于这个sayName方法对于所有person都是一致的，如果我们定义在对象里面会非常麻烦，所以就有了原型

原型是什么

无论什么时候只要创建了一个函数，就会根据一组特定的规则为该函数创建一个prototype属性，这个属性指向函数的原型对象，同时呢，这个原型对象会有个constructor属性，这个属性指向函数对象，即Person.prototype.constructor指向person

这里有个注意点： 当用构造函数创建了一个新实例后，该实例也有一个内部属性叫[[Prototype]],在浏览器实现中叫_ proto _ ,这个属性是对象和原型之间的连接，而不像prototype那个是构造函数与原型间的连接。

  1. 可以通过Object.getPrototypeOf()来获取对象的原型
  2. 可以通过person.hasOwnPrototype()来判断某个属性是来源于对象还是原型

Javascript中的逻辑运算

逻辑运算符通常用于布尔型（逻辑）值；这种情况，它们返回一个布尔型值。然而，&&和||运算符实际上返回一个指定操作数的值，因此这些运算符也用于非布尔型，它们返回一个非布尔型值。

运算符 示例 说明
逻辑与 (&&) expr1&&expr2 如果expr1 能转换成false则返回expr1,否则返回expr2. 因此, 在Boolean环境中使用时, 两个操作结果都为true时返回true,否则返回false.
逻辑或 (` `) expr1\ \ expr2

经常能看到javascript的逻辑运算用在变量初始化的时候，之前一直不是很理解，虽然知道有惰性求值这个概念，但是因为以前学java的时候只知道逻辑运算符只能用在求布尔值的时候，也就是这个表格里说的boolean环境中，js的逻辑运算还可以用在非boolean的环境
