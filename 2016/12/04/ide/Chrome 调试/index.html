Chrome调试

发表于 2016-12-04   |   分类于 ide

https://egghead.io/lessons/tools-unbundle-your-javascript-with-source-maps-in-chrome-devtools

  1. 在Chrome 开发者工具的debugger可以加断点调试，F10和F11分别表示setp over to the next function call 和step into the next function call，watch用来检测表达式
    1. 如何调试webpack打包后的源码呢呢？ 因为webpack会把所有的依赖以至于代码看起来会非常混乱，所以调试的时候需要分清楚webpack的源码，方法就是在webpack打包的的module.exports的对象里加上这么一条 devtool:”#eval-source-map”
  2. 在source面板下有个snippet，可以在里面写自动化测试函数来简化测试工作
#chrome debug
