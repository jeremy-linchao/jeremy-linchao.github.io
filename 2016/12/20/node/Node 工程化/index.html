Node 工程化

  1. 目录结构：模块应用遵循CommanJS，web应用通过框架或者自行按MV*组织

  2. 构建工具：make(需要bash编程)，Grunt(本身用nodejs写)

  3. 编码工具：JSLint和JSHint，一旦团队约定了编码规范的详细细则，则可以生成一份规则文件，扫描工具或者编辑器能够通过该规则文件对源码进行扫描

  4. 代码审查：1）功能点是否正确完成 2）编码风格是否符合规范 3）单元测试是否有同步添加

  5. 部署环境：代码 -> stage(测试环境) -> 预发布 -> 生产，之所以有个预发布环境是因为测试环境只能满足功能的正确性，而预发布环境可以做数据相关的检查，它的数据较为接近线上真实的数据

  6. 部署操作：nohup node app.js & ，停止和重启是通过相应的pid写进app.id文件下

  7. 日志记录：通常的API，尽量不要隐藏错误，不要通过try/catc捕获异常，事实上，日志通常是服务于业务的，尽量由最上层的调用者捕获记录，底层调用或中间层调用出现的异常只要正常传递给上层的调用方即可
