git的进一步理解

发表于 2017-11-14   |   分类于 git

  1. git add： 这个命令目标文件的状态不同会用不同作用 1）已经跟踪的文件: 把目标文件加到暂存区 2）没有跟踪过的文件: 标记为需要跟踪并加到暂存区 3）合并时有冲突： 标记为已解决

  2. .gitignore:

    比如写node_modules/只会忽略当前node_modules下的所有文件，但不会忽略子目录下的node_modules，所以如果对于有多个子项目，最好在每个子项目下分别写gitignore

  3. git diff 比较的是什么？ 是工作目录中当前文件和暂存区域快照之间的区别

  4. git diff - - staged或git diff - - cached 比较的是什么？已经暂存起来的文件和上次提交时的快照之间的区别

  5. git rm： git rm 跟git add是同级别的命令，意味这个git rm之后也可以直接git commit，它的作用与 git add正好相反，它是从暂存区移除掉某个文件，不加入到版本管理

kubernetes集群搭建

发表于 2017-10-30   |   分类于 kubernetes

follow的文档主要是这个

https://k8s-install.opsnull.com/

有几个问题要注意

  1. flannel在部署完之后是不能ping通网关的，因为这时候docker根本没有起来，按着文档的章节，docker在node那一节，因此这里ping不通是正常的，只要在docker安装完之后可以ping就行

  2. dashboard: 文档给的service部署yaml没有问题，但是nodePort没有指定，所以是段内一个随机的，如果要提供外网访问，一般起一个nginx反代，所以最好也指定nodePort, dashboard-service.yaml如下

    http {
    log_format main ‘$remote_addr - $remote_user [$time_local] “$request” ‘

    '$status $body_bytes_sent "$http_referer" '
'"$http_user_agent" "$http_x_forwarded_for"';

    access_log /var/log/nginx/access.log main;

    sendfile on;
    tcp_nopush on;
    tcp_nodelay on;
    keepalive_timeout 65;
    types_hash_max_size 2048;

    include /etc/nginx/mime.types;
    default_type application/octet-stream;

    Load modular configuration files from the /etc/nginx/conf.d directory.

    See http://nginx.org/en/docs/ngx_core_module.html#include

    for more information.

    include /etc/nginx/conf.d/*.conf;

    upstream dashboard {

    server 10.146.0.2:8900 weight=1;
 server 10.146.0.3:8900 weight=1;
 server 10.146.0.4:8900 weight=1;

    }

    server {

    listen       80 default_server;
listen       [::]:80 default_server;
server_name  YOUR_SERVER;

# Load configuration files for the default server block.
include /etc/nginx/default.d/*.conf;

location / {
    proxy_pass  http://dashboard;
}

error_page 404 /404.html;
    location = /40x.html {
}

error_page 500 502 503 504 /50x.html;
    location = /50x.html {
}

    }

另外在centos 7上，如果访问还能看到这个nginx有这个错误：Permission denied) while connecting to upstream, 原因是开了SELinux，需要关闭它

1.临时关闭（不用重启机器）：

setenforce 0 ##设置SELinux 成为permissive模式

##setenforce 1 设置SELinux 成为enforcing模式

2.修改配置文件需要重启机器：

修改/etc/selinux/config 文件

将SELINUX=enforcing改为SELINUX=disabled

重启机器即可
