关于sql连接的理解

发表于 2018-01-01   |   分类于 sql
  1. 内连接： 内连接就是关系代数里面的θ-连接，所谓θ连接就是先做笛卡尔乘积，然后在结果上做一个选择操作，条件就是θ指定的条件，比如某两列大小比较
  2. 等值连接：等值连接就是特殊的内连接，特殊在θ条件变为两列相等，但等值连接最后的列数还是和笛卡尔乘积的列数一样
  3. 自然连接：自然理解是再进一步特殊的等值连接，自然连接有个前提条件是连接的两张表必须要有相同的列名，然后再做一次等值连接，区别说自然连接会去除掉重复的列
  4. 外连接：为什么要有外连接呢？因为在做等值连接的时候可能会丢失掉某些行的信息，比如第一个表的东西，比如顾客表和订单表的例子，要查询所有顾客的所有订单，自然连接是不够的，因为可能有些顾客没下过订单，因此外链接就出来了，它在连接的时候如果发现订单表没有顾客数据，就填充0。至于左外连接和右外连接还是全外连接，则取决于你是要哪边的数据保留完整

why I love go

发表于 2017-12-28   |   分类于 go
  1. gofmt: 终于不用一个guide来告诉你有什么风格了，不会像js那样讨论需不需要有;了, 不用再争论是两个还是四个空格这些问题了
  2. defer函数: 对于资源的关闭，回收处理使用起来太方便了，如果你写过sql的事务你应该能体会出错时到处rollback的郁闷
  3. 一个for循环能做掉所有事，不用再有js的for in; for of，一个for就能遍历数组，map, string
  4. 变量短声明：类似于c++的auto，让这门静态语言有了类似动态语言的变量定义语法
  5. goroutine：一开始对于goroutine的channel玩法不是很理解，什么buffer的，unbuffered channel会感到很陌生，但是当使用多了之后发现并发编程在go里真的很简单
  6. 匿名函数与闭包：对于函数式编程能做到一部分支持

protocol buffer安装

  1. protocol buffer安装分为两部分：一个是protocol编译器的安装，编译器主要是将proto文件编译成相应语言可用的代码，当然这个也需要相应语言插件支持，编译器是去到https://github.com/google/protobuf/releases 下载一个对应平台的zip包； 二是protocol buffer 运行时环境安装，运行时是什么呢，就是语言来调用protocol, 比如go要序列化，调用相应函数，而这个函数就是运行时环境来提供的，对于go来说就是protoc-gen-go这个软件，这个软件提供protocol的编译器插件以及运行时环境

  1. what: protocol buffer是什么?

    protocol buffer是一种序列化结构型数据机制，就像xml，json, 但protocol buffer不仅仅只是消息交换机制，它还是IDL，定义了接口

  2. how: protocol buffer怎么工作?

    1) 定义protocol文件

    2） 使用特定语言的protocol buffer编译器来编译.proto文件，这样会生成特定的类以及访问和特定语言文件，比如上面那个例子，如果用的是c++，就会生成一个.cc的proto文件，里面会很有class和get/set方法，你就可以使用下面的方法

    3） why: 为什么用protocol而不是xml或json

    ​ 1) protocol 序列化的数据更小 2）解析更快 ，快20到100倍 3）可以生成数据访问类代码

    ​ 当然xml也有优势，比如xml是可读可编辑并且在某种程度上可以自我描述的

  3. what: 什么是gRPC

    gRPC是一个rpc框架，client可以调server的方法就像本地调用一样，远端的server与client可以用完全不同的语言来实现。这里面涉及到几个问题： 1）怎么实现远程调用？因为你本地函数调用使用的是内存的数据，即怎么做序列化/反序列化，用什么数据格式？现有的数据格式是xml或json，而gRPC默认使用的是protocol buffer，优点前面也说了，报文更小，解析更快。 2） 第二个问题是既然涉及到了多语言，那么必然需要有涉及到的多语言的解析库和序列化库，这个库我更愿意理解为一种sdk，像json，xml许多语言都提供序列化库了，比如js中的JSON.stringify和JSON.parse，3）rpc需要让客户端知道有哪些方法可以调吧，就像浏览器和服务端通信用http/json一般是给出api文档，表示方法是什么，入参是什么，返回值是什么，在gRPC或者rpc框架中一般是用service来定义，这个是rpc的概念

  4. gRPC类型：

    1）Unary RPC 2）Server streaming RPC 3) Client streaming RPCs 4)Bidirectional streaming RPCs
