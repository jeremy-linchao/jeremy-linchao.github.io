对于cookie和session的理解

发表于 2016-12-27   |   分类于 concept
  1. 首先cookie是后端服务器通过response响应传送给浏览器的一个字段，只要不过期，之后浏览器在每次访问的时候就会加上这个字段，在nodejs里，cookie被封装在req.headers.cookie，是个字符串，可以写一个解析函数来转换成js对象
  1. 服务器是通过Set-Cookie这个字段来发送cookie的，最重要的是name=value, 可选项包括Path, Expires,Max-age,httpOnly,Domain,Secure
  2. Cookie对性能会有影响，表现在如果浏览器会将cooke放到每次请求上，然而对于静态页面，其实是不需要状态的，因此把静态页面用独立域名会减少cookie的传输
  3. 浏览器通过document.cookie把cookie暴露给了js，因此如果你在cookie写些重要的关于业务信息，很容易被篡改

函数式编程

发表于 2016-12-21   |   分类于 coding

有些事物在你得到之前是无足轻重的，得到之后就不可或缺了

  1. 纯函数：相同的输入，永远得到相同的输出，而且没有任何可观察的副作用
  2. 副作用：在计算结果的过程中，系统状态的一种变化，或者与外部世界进行可观察的交互
  3. 函数的柯里化：只传递给函数一部分参数来调用它，让它返回一个函数去处理剩下的参数
  4. currying : the technique of translating the evaluation of a function that takes multiple arguments into evaluating a sequence of functions, each with a single argument.
  5. partial application : the process of fixing a number of arguments to a function, producing another function of smaller arity
  6. 柯里化和偏函数的区别：区别在于固定的参数不同，柯里化每次固定一个参数，返回一系列一元函数的调用

关于异步

发表于 2016-12-21   |   分类于 node

为什么需要异步？

  1. 前端无疑是异步用的最多的地方，举个例子，浏览器中javascript的执行与UI渲染是在同一个线程，这样的话如果javascript如果同步去获取远端一个资源，在这个期间UI的渲染和响应就是停止的，用户点击任何东西是没有反应的，用户体验非常差，通过异步的方式可以消除这个阻塞现象

  2. 在内核IO调用只有阻塞与非阻塞，在调用阻塞IO时，应用程序需要等待I/O完成才返回结果；而非阻塞I/O则不带数据直接返回，要获取数据，还需要通过文件描述符再次读取

  3. 前面提到了为了获取完整的数据，应用程序需要重复调用I/O操作来确实是否完成，这种重复调用判断操作是否完成的技术叫做轮询，有以下几种形式：

    a) read b)select c)epoll 4) 这些个技术对于应用程序而言，仍然只能算是一种同步，因为应用程序需要等待I/O完全返回，等待期间，CPU要么用于遍历文件描述符的状态，要么用于休眠等待时间发生

  4. Linux下面存在一种异步IO方式叫AIO，但这个只在linux下有，并且仅支持O_Direct方式读取

  5. 现实场景下的异步IO： 线程池模拟异步I/O

    一个线程进行计算处理，让其他线程进行非阻塞I/O加轮询获取数据，通过线程间通信将I/O的数据进行传递

  6. 我们说node是单线程的，这里的单线程仅仅只是javascript执行在单线程中，内部完成I/O任务的另有线程池

ES6 Generate 函数

发表于 2016-12-20   |   分类于 coding

  1. generator函数可以用来做一种异步编程解决方案，generator函数有多种理解角度，从语法上generator函数是一个状态机，封装了多个内部状态。它返回一个遍历器对象，所以generator函数除了是个状态机，还是一个遍历器对象生成函数

  2. generator函数是一个普通函数，但有两个特征，1）function关键字与函数名之间有一个星号 2)函数体内部使用yield语句，定义不同的内部状态

  3. 调用Generator函数，返回一个遍历器对象，代表Generator函数的内部指针。以后，每次调用遍历器对象next方法，就会返回一个有着value和done两个属性的对象。value属性表示当前的内部状态的值，是yield语句后面那个表达式的值；done属性是一个布尔值，表示是否遍历结束

  4. yield句本身没有返回值，或者说总是返回undefined。next方法可以带一个参数，该参数就会被当作上一个yield语句的返回值
