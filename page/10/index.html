现代javacript

发表于 2017-03-08

  1. nodejs： nodejs是一个javascript运行时环境，可以类比于java的虚拟机，在后端的作用比较好理解，业务逻辑开发，工具脚本开发等都是它的用武之地，但是在前端开发中也会经常包含node，这点有时候不是太好理解。nodejs在前端的作用主要有如下几个： 前端程序的打包，测试，以及语法规范检查等等

  2. npm： node默认的包管理器

  3. yarn： 新的node包管理器，速度更快，离线support，并且获取依赖会更快

  4. package.json: 项目的描述和配置文件，包括项目名称，版本，贡献者，依赖等等，除了可以用npm init之外，也可以用yarn init来生成

  5. babel-node的作用：replace your call to the node binary and execute ES6 files directly on the fly，之前看到这句话不是很理解，node不是对ES6支持的非常好吗，为什么还需要babel呢？回想一下，有一个语法是至少现在的node是不支持的，那就是模块。node的模块化系统用的是COMMONJS标准，而ES6用的是import和export，这里就需要babel-node来做转化了

  6. ES-lint: ESLint is the linter of choice for ES6 code. A linter gives you recommendations about code formatting, which enforces style consistency in your code, and code you share with your team. It’s also a great way to learn about JavaScript by making mistakes that ESLint will catch.

    我的理解：ES-lint就是一种代码格式化的规范，有了他团队协作开发就很方便，之前可能还需要在code_review里做些代码格式的检查，那现在只需要选一种规范，加载到ES-lint去，团队每个成员的代码都能自动遵循某个规范了，这里面可以用airbnb的规范

    另外ESlint可以集成到IDE里去，不要用IDE本身的lint，因为那样的话无法达成统一的规范

  7. Flow: A static type checker by Facebook. It detects inconsistent types in your code. For instance, it will give you an error if you try to use a string where should be using a number.

    如果说ES-lint是代码格式检查的工具，那Flow就是对javascript不是强类型作规范

    yarn add –dev flow-bin babel-preset-flow babel-eslint eslint-plugin-flowtype

    flow-bin is the binary to run Flow in our scripts tasks, babel-preset-flow is the preset for Babel to understand Flow annotations, babel-eslint is a package to enable ESLint to rely on Babel’s parser instead of its own, and eslint-plugin-flowtype is an ESLint plugin to lint Flow annotations

  8. Jest: 一个facebook的测试库，可以测试react

  9. Husky： git钩子， 可以在git commit和git push之前运行脚本测试代码
