Javascript的prototype

发表于 2016-12-17   |   分类于 javascript

为什么要有原型

考虑一个需要定义一个person对象

这个person会有名字、年龄和工作的属性，这些都是每个person对象不一样的地方，但是如果我们要定义一个sayName方法，由于这个sayName方法对于所有person都是一致的，如果我们定义在对象里面会非常麻烦，所以就有了原型

原型是什么

无论什么时候只要创建了一个函数，就会根据一组特定的规则为该函数创建一个prototype属性，这个属性指向函数的原型对象，同时呢，这个原型对象会有个constructor属性，这个属性指向函数对象，即Person.prototype.constructor指向person

这里有个注意点： 当用构造函数创建了一个新实例后，该实例也有一个内部属性叫[[Prototype]],在浏览器实现中叫_ proto _ ,这个属性是对象和原型之间的连接，而不像prototype那个是构造函数与原型间的连接。

  1. 可以通过Object.getPrototypeOf()来获取对象的原型
  2. 可以通过person.hasOwnPrototype()来判断某个属性是来源于对象还是原型

javascript中的逻辑运算

发表于 2016-12-16   |   分类于 javascript

逻辑运算符通常用于布尔型（逻辑）值；这种情况，它们返回一个布尔型值。然而，&&和||运算符实际上返回一个指定操作数的值，因此这些运算符也用于非布尔型，它们返回一个非布尔型值

逻辑与： 在非boolean环境下，如果expr1能转换成false则返回expr1,否则返回expr2.
逻辑或： 在非boolean环境下，如果expr1能转换成true则返回expr1,否则返回expr2.

经常能看到javascript的逻辑运算用在变量初始化的时候，之前一直不是很理解，虽然知道有惰性求值这个概念，但是因为以前学java的时候只知道逻辑运算符只能用在求布尔值的时候，也就是这个表格里说的boolean环境中，js的逻辑运算还可以用在非boolean的环境

API设计

发表于 2016-12-14   |   分类于 concept

  • 所有API请求都使用https

  • 在response的accept header上加一个版本号，比如 version = 3

  • 在所有的response里支持ETags，这样可以允许用户来缓存资源

  • 在每个header中包含一个request-id，由uuid构成，目的是为了方便调试和追踪

  • 大的response应该用Range头来指定如何获取这些数据
    ​

    Request

    1. post，put等这些请求的body中只接受序列化的json格式，不要用类似form-endoded的方式，这样可以保持请求和返回的一致
    2. 尽量用复数（加s），除非那个资源是个单实例
    3. 对于每个独立资源，尽量不使用任何special actions，如果需要action，也放在单独的action下 /resources/:resource/actions/:action
      1. 使用小写字母，对于路径名使用这个划线 “-“，对于属性名，使用下划线_
      2. 路径长度最小化

        Response

  1. 返回合适的状态码

    200： GET，POST，DELETE，PATCH同步操作顺利完成

    201：POST，PUT异步创建资源成功

    202：POST，PUT，DELETE异步操作

    206：GET操作成功，但只返回一部分数据

    401 Unauthorized： 鉴权失败

    403 Forbidden：对于某一个资源没有特别的权限

    422：无效参数

    429: 请求次数太频繁，做速率限制

    500：内部服务器错误

  2. 对于每一个资源提供一个UUID

  3. 对于每个资源提供时间戳

  4. 使用UTC times ISO8601

  5. 使用嵌入对象做外键引用
