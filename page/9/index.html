利用git来阅读源代码

发表于 2017-03-21   |   分类于 git

当需要看一个开源库的源代码的时候，比较好的做法是从他比较早的版本来看，如果时间充裕，也可以直接从第一版开始看，这里就需要比较多的git切换操作，有几个命令会很常用，这里列一下

  1. git log - - pretty=oneline : 这个命令用于列出到当前head位置所有的历史提交
  2. 版本切换 git checkout 5dg322fasf 可以加 -f 来强制切换，在某些情况自己改了某个版本的文件情况下会有用
  3. git reflog: 查看git的操作记录， 因为git log只是列出到当前commit的历史提交，一旦checkout到较早的commit，想回到较早的版本得找到最近一次提交，git reflog提供了所有关于版本移动的历史，可以通过这个来找到最原始的commit号，当然github本身网站上也有这些信息，只是那样会慢很多还需要网络，不是太方便

关于fail2ban

看到日志里有很多如下的错误，确定是暴力破解，使用fail2ban来防止暴力破解

具体的步骤如下：

  1. 切换root： sudo su -

  2. 安装fail2ban sudo apt-get install fail2ban

  3. 创建fail2ban相应的ss顾虑器，可以进到/etc/fail2ban/filter.d/，看到有很多创模板过滤器，vim shadowsocks.conf

  4. 修改默认的fail2ban配置，vim /etc/fail2ban/jail.conf ，将banaction改成iptables

  5. 添加jail.local配置， vim /etc/fail2ban/

  6. 启动服务 service fail2ban start

  7. 查看是否生效： fail2ban-client status shadowsocks
