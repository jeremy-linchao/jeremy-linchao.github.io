mac上的shadowsocks

发表于 2017-01-12   |   分类于 utility

  1. 首先提一下，mac上的shadowsocks会比windows上的用起来简单一些，mac的不用配代理，原因在于mac下的chrome代理是跟着系统走的，firefox需要手动配置下代理类型，当然也可以跟系统走，但是国内的版本不行

  2. 全局模式和自动代理模式区别，开启全局模式下会把系统设置为socks代理，开启代理模式会把系统设置为pac代理，shadowsocks的pac代理服务器起在8090端口

  3. pac和switchyproxy区别： pac是一个js文件，从shadowsocks起的8090端口的服务器获得，告诉浏览器该怎么找到代理服务器，而switchyproxy则是一个浏览器插件，运行在浏览器端

  4. 关于dns是否是远程解析的验证方式,开一个wireshark，设定过滤dns，结论

    1. chrome不管是pac还是socks5都是要remote dns 解析
    2. safari也是远程解析
  5. safari也是走的系统默认代理，理论上都能翻，但实际上facebook和twitter大多可以，google基本上不去，怀疑是浏览器问题，不过50.1.0版的firfox 真是垃圾，国际版也是，无论哪种模式都翻不出去

皮萨诺周期

先解释下皮萨诺周期是什么，我们知道斐波那契数列的通项公式是 F(n) = F(n-1) + F(n-2)，其中F(0) = 0, F(1) = 1,F(n) 对于m求模得到的序列是个周期序列，但周期本身目前还没有计算公式可以得出。现在有个问题要求大斐波那契数列对于m的求余值，为题具体描述如下：

由于n太大，计算Fn显然不现实，一来循环的次数很多，更重要的是没有方法来存这个数值，要解决这个问题有几个关键点

1）设定P(n)是F(n) % m，P(n)是个周期序列，且每个P(n)的周期都是以数字0 1开头

2）(a+b)%m = (a%m + b%m)%m

3) 周期的最大值为6*m

有了这两点基本知识，大体思路就出来了，首先用循环去计算周期，凡是循环到0 1就认为一个周期结束，因为m的最大值只有10的5次方，所以循环最多的次数也就只需要6 （10 * 5），这个在1s内是可以完成的，第二个就是计算余数的周期序列，这就要利用性质2了，附一份c++的代码

  1. 默认状态下，const对象仅在文件内有效

    编译器在编译过程会找到所有用到bufSize的地方替换成相应的值。如果程序包含多个文件，则每个用了const对象的文件都必须能访问到它的初始值，所以在每一个用到变量的文件中都要有对它的定义。为了支持这个用法，默认情况下const对象被设定为仅在文件内有效，当多个文件中出现了同名const变量，其实等同于在不同文件中分别定义了独立的变量

  2. 如果想在多个文件之间共享const对象，必须在变量的定义之前添加extern关键字

Generator

发表于 2016-12-29   |   分类于 ES6

  1. 定义:

  2. 调用： var gen = idMaker(); 调用一个生成器函数并不马上执行它的主体，而是返回一个这个生成器函数的迭代器（iterator）对象。

  3. 使用：该表达式定义了迭代器返回的值，或者，被 yield委派至另一个生成器函数。next()方法返回一个对象，该对象有一个 value属性，表示产出的值，和一个done属性，表示生成器是否已经产出了它最后的值

    next方法的作用是分阶段执行Generator函数。每次调用next方法，会返回一个对象，表示当前阶段的信息（value属性和done属性）。value属性是yield语句后面表达式的值，表示当前阶段的值；done属性是一个布尔值，表示Generator函数是否执行完毕，即是否还有下一个阶段
