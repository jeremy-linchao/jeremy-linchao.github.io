npm配置依赖

发表于 2016-12-28   |   分类于 node
  • 指定版本：比如1.2.2，遵循“大版本.次要版本.小版本”的格式规定，安装时只安装指定版本。
  • 波浪号（tilde）+指定版本：比如~1.2.2，表示安装1.2.x的最新版本（不低于1.2.2），但是不安装1.3.x，也就是说安装时不改变大版本号和次要版本号。
  • 插入号（caret）+指定版本：比如ˆ1.2.2，表示安装1.x.x的最新版本（不低于1.2.2），但是不安装2.x.x，也就是说安装时不改变大版本号。需要注意的是，如果大版本号为0，则插入号的行为与波浪号相同，这是因为此时处于开发阶段，即使是次要版本号变动，也可能带来程序的不兼容。
  • latest：安装最新版本。

对于cookie和session的理解

发表于 2016-12-27   |   分类于 concept
  1. 首先cookie是后端服务器通过response响应传送给浏览器的一个字段，只要不过期，之后浏览器在每次访问的时候就会加上这个字段，在nodejs里，cookie被封装在req.headers.cookie，是个字符串，可以写一个解析函数来转换成js对象
  1. 服务器是通过Set-Cookie这个字段来发送cookie的，最重要的是name=value, 可选项包括Path, Expires,Max-age,httpOnly,Domain,Secure
  2. Cookie对性能会有影响，表现在如果浏览器会将cooke放到每次请求上，然而对于静态页面，其实是不需要状态的，因此把静态页面用独立域名会减少cookie的传输
  3. 浏览器通过document.cookie把cookie暴露给了js，因此如果你在cookie写些重要的关于业务信息，很容易被篡改

函数式编程

发表于 2016-12-21   |   分类于 coding

有些事物在你得到之前是无足轻重的，得到之后就不可或缺了

  1. 纯函数：相同的输入，永远得到相同的输出，而且没有任何可观察的副作用
  2. 副作用：在计算结果的过程中，系统状态的一种变化，或者与外部世界进行可观察的交互
  3. 函数的柯里化：只传递给函数一部分参数来调用它，让它返回一个函数去处理剩下的参数
  4. currying : the technique of translating the evaluation of a function that takes multiple arguments into evaluating a sequence of functions, each with a single argument.
  5. partial application : the process of fixing a number of arguments to a function, producing another function of smaller arity
  6. 柯里化和偏函数的区别：区别在于固定的参数不同，柯里化每次固定一个参数，返回一系列一元函数的调用
