typescript学习

发表于 2017-03-23   |   分类于 typescript

typescript是javascript的超集，那超在哪儿？

1. 基本类型增强
  1. tuple： 长度固定的数组，但里面的值不需要是同一个类型

如果访问超过了tuple的索引范围，就会使用union type

  1. 枚举类型： 所谓枚举无非是给数值类型起一个比较友好的名字罢了，提高开发效率，比如通信状态的返回值可能有很多，用枚举远比数字类型来的直观，没有枚举的话就需要使用对象了
  1. any类型: 当不确定某个值的类型时可以用any来通过编译
  2. void类型：没有类型，多用于函数返回值
  1. never类型：永远不会发生的类型，什么情况下会有用？ 如果一个函数就是抛一个错，那么它的返回值就可以是never
  1. 类型断言：当我们确定某个值是什么类型的时候，像是其他语言里的类型转换，但是不会做类型检查和转换，也不会有运行时的影响，只是单独给编译器使用
2. 类型声明

  1. var的作用域会泄露到包含它的函数或者包含它的模块

  2. 在同一个函数中var声明的变量可以重复声明多次

  3. 在循环里声明的let和普通的let有很大区别，循环里的let在每次递归时都会创建一个独立的作用域， 再看下这两段经典的代码：

3. interface
  1. interface是ts的一个核心，它主要用来检查变量值的类型，因为javascript的变量的类型是可变的，实际上是它的值的类型是可变的，interface起到这个作用，它来定义某个变量可以有的类型，因此在编译阶段就能发现此类错误
4. 可索引的类型

​ 可以用字符串或者数字来作索引，但是数字索引的返回值必须是字符串索引返回值类型的子类型。 这是因为当使用number来索引时，JavaScript会将它转换成string然后再去索引对象。

  2. console.log()此类的函数不是ECMA的语法，而是宿主环境提供的，所以不同的宿主环境下他们的表现可能不同，在某些宿主环境可能会把console.log()这些异步操作，如果真要看一个值得话，最好使用debug以及断点

  3. setTimeout()只是设置一个定时器，在指定的时间到之后把回调函数放到事件队列里去，但是如果这个事件队列里还有其他的事件，这个回调函数还需要等其他事件执行完，所以setTimeout()能保证一定在多少时间之后执行，但是不一定在那个时间一定执行。 事件循环可以参考这个伪代码

利用git来阅读源代码

发表于 2017-03-21   |   分类于 git

当需要看一个开源库的源代码的时候，比较好的做法是从他比较早的版本来看，如果时间充裕，也可以直接从第一版开始看，这里就需要比较多的git切换操作，有几个命令会很常用，这里列一下

  1. git log - - pretty=oneline : 这个命令用于列出到当前head位置所有的历史提交
  2. 版本切换 git checkout 5dg322fasf 可以加 -f 来强制切换，在某些情况自己改了某个版本的文件情况下会有用
  3. git reflog: 查看git的操作记录， 因为git log只是列出到当前commit的历史提交，一旦checkout到较早的commit，想回到较早的版本得找到最近一次提交，git reflog提供了所有关于版本移动的历史，可以通过这个来找到最原始的commit号，当然github本身网站上也有这些信息，只是那样会慢很多还需要网络，不是太方便

关于fail2ban

看到日志里有很多如下的错误，确定是暴力破解，使用fail2ban来防止暴力破解

具体的步骤如下：

  1. 切换root： sudo su -

  2. 安装fail2ban sudo apt-get install fail2ban

  3. 创建fail2ban相应的ss顾虑器，可以进到/etc/fail2ban/filter.d/，看到有很多创模板过滤器，vim shadowsocks.conf

  4. 修改默认的fail2ban配置，vim /etc/fail2ban/jail.conf ，将banaction改成iptables

  5. 添加jail.local配置， vim /etc/fail2ban/

  6. 启动服务 service fail2ban start

  7. 查看是否生效： fail2ban-client status shadowsocks
