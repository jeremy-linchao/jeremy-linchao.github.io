css

发表于 2017-02-05   |   分类于 html/css

  1. css也可以注释，使用/ /

  2. color代表前景色，background-color代表背景色

  3. css中对同一个选择器的多个声明，最后一个生效，这个特点可以用来作浏览器兼容，比如在color上，第一行写一个所有浏览器都兼容的color表示法，下面一行写一个较新的表示法

  4. 一个html元素可以加上多个css类，但最好限制在4个最多

  5. 盒模型的width和height

    1) 用像素表示： 表示盒子的精确大小

    2）ems: 相对于盒子里面文字的倍数

    3）百分数：相对于父盒子的比例

  6. 当margin:auto被用来做居中时，必须同时加上width这个属性用来标明该元素的宽度，否则这个div的宽度就被设置成它父元素的宽度了，也就没有居中的说法了

  7. display属性有如下几个： inline,block,incline-block,none

  8. 默认盒模型有一个缺点，就是padding和border会增加width，因此无法计算每个元素的实际大小

  9. 盒模型使用box-sizing这个属性指定的，默认值为content-box，这个模型的width并不是实际渲染的大小，实际元素为加上boder的宽度以及pading，为了解决这个问题可以把box-sizing改为border-box，在这个模型下，width就是实际渲染出来的大小，padding和border会被相应地减掉

  10. position用来指定元素的位置，有如下四个取值

    1）static： 默认值

    2）relative：这个将元素设定为相对，因此还需要指定top,left,bottom和right属性，表示相对于四周的像素值

    3）absolute: 这个元素表示该元素就被固定在那边，其他元素会将他视而不见

    4) fixed: 会固定住这个元素，即使在有滚动条的情况下

  11. 当一个元素是float时，在同一个父元素下的其他元素会靠着它流动起来，同时float的元素必须指定宽度

react学习

发表于 2017-01-17   |   分类于 react

正如研究任何一种框架一样，研究react也要搞清楚作者开发他的动机，其实说白了就是前段组件化，这样的好处是可以复用很多代码，不仅仅是自己写的组件，别人的组件也可以，这是不是很酷呢？

既然是组件，那组件势必要把view显示和逻辑封装在一起，这个就和很多之前的前端MVC开发方式有很多不同，是需要适应的，但我觉得组件这个东西并不新颖，在UI开发的历史上早就存在

既然是组件，有几个概念很重要，compenent不用多说，那如何把数据传到component中呢，比如我有一个表格需要显示数据，怎么把数据传进来呢？组件是怎么来处理css的呢，毕竟组件需要提供很多不同显示样式？这些都是需要带着的问题去看react的

  1. prop和state的区别，他们是用来干嘛的? 这里有句话：State is similar to props, but it is private and fully controlled by the component. 显然他的意思是说state是组件自有的属性，完全私有。那么它用来做什么呢？举个例子，有个时钟的组件，时钟每秒钟都得更新下，如果用props是不可以的，因为props传进来是不能改变的，而state是可变的，同时对于时钟来说，时间这个属性本来就是完全它自己的，不需要外面传数据进来。

  2. react的数据是单向绑定，state只能用setState函数来修改(除了构造函数)，state的更新可能是异步的

  3. react的事件命名是驼峰的，事件处理器本身是一个JSX表达式

    https://www.smashingmagazine.com/2014/01/understanding-javascript-function-prototype-bind/

    js类里的成员函数默认是不绑定的，这篇文章解释的比较清楚

  4. react-router如何与ant-design的组件结合?

    有一个问题一直想不通，就是像ant-design的menu组件不是都有超链接了嘛，router这个link又是用来干嘛呢？这里一定要搞清楚一个问题，在menu组件里的连接可以认为就是带了标签的跳转，但是跳转到哪儿去是有react-router里的各条route来决定的，他们之间并不冲突

  5. Components must return a single root element，所以return的element必须加一个包裹

  6. All React components must act like pure functions with respect to their props

  7. 当把函数式组件改为class组件时，有两点需要注意，1）多加了个render函数，里面定义个return函数，这其实是个闭包 2）不需要写函数参数props，反而是在函数需要用到props的时候写this.props.date

  8. While this.props is set up by React itself and this.state has a special meaning, you are free to add additional fields to the class manually if you need to store something that is not used for the visual output.

    If you don’t use something in render(), it shouldn’t be in the state.

  9. 关于setState的三个注意点： 1）所有修改state数据只能用this.setState方法，换句话说除了在构造函数能访通过this.state=的形式来设置state，其余地方都只能用函数，但是访问是没有问题的 2）states的更新可能是异步的，React may batch multiple setState() calls into a single update for performance. 要更新的话需要用setState的第二种用法，参数传一个回调函数 3）states的更新是合并在一起的，但不同的setState不会影响各自

  10. 事件处理一般是类的成员函数，但是需要在构造函数里绑定this或者成员函数写成=的形式

  1. 首先提一下，mac上的shadowsocks会比windows上的用起来简单一些，mac的不用配代理，原因在于mac下的chrome代理是跟着系统走的，firefox需要手动配置下代理类型，当然也可以跟系统走，但是国内的版本不行

  2. 全局模式和自动代理模式区别，开启全局模式下会把系统设置为socks代理，开启代理模式会把系统设置为pac代理，shadowsocks的pac代理服务器起在8090端口

  3. pac和switchyproxy区别： pac是一个js文件，从shadowsocks起的8090端口的服务器获得，告诉浏览器该怎么找到代理服务器，而switchyproxy则是一个浏览器插件，运行在浏览器端

  4. 关于dns是否是远程解析的验证方式,开一个wireshark，设定过滤dns，结论

    1. chrome不管是pac还是socks5都是要remote dns 解析
    2. safari也是远程解析
  5. safari也是走的系统默认代理，理论上都能翻，但实际上facebook和twitter大多可以，google基本上不去，怀疑是浏览器问题，不过50.1.0版的firfox 真是垃圾，国际版也是，无论哪种模式都翻不出去

皮萨诺周期

先解释下皮萨诺周期是什么，我们知道斐波那契数列的通项公式是 F(n) = F(n-1) + F(n-2)，其中F(0) = 0, F(1) = 1,F(n) 对于m求模得到的序列是个周期序列，但周期本身目前还没有计算公式可以得出。现在有个问题要求大斐波那契数列对于m的求余值，为题具体描述如下：

由于n太大，计算Fn显然不现实，一来循环的次数很多，更重要的是没有方法来存这个数值，要解决这个问题有几个关键点

1）设定P(n)是F(n) % m，P(n)是个周期序列，且每个P(n)的周期都是以数字0 1开头

2）(a+b)%m = (a%m + b%m)%m

3) 周期的最大值为6*m

有了这两点基本知识，大体思路就出来了，首先用循环去计算周期，凡是循环到0 1就认为一个周期结束，因为m的最大值只有10的5次方，所以循环最多的次数也就只需要6 （10 * 5），这个在1s内是可以完成的，第二个就是计算余数的周期序列，这就要利用性质2了，附一份c++的代码

c++ const

发表于 2017-01-04   |   分类于 c++

  1. 默认状态下，const对象仅在文件内有效

    编译器在编译过程会找到所有用到bufSize的地方替换成相应的值。如果程序包含多个文件，则每个用了const对象的文件都必须能访问到它的初始值，所以在每一个用到变量的文件中都要有对它的定义。为了支持这个用法，默认情况下const对象被设定为仅在文件内有效，当多个文件中出现了同名const变量，其实等同于在不同文件中分别定义了独立的变量

  2. 如果想在多个文件之间共享const对象，必须在变量的定义之前添加extern关键字
