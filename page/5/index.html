发表于 2017-02-09   |   分类于 javascript

在初次看《javascript高级程序设计》的时候，遇到apply，call，bind这一节的时候也是迷糊的跳过了，但之后发现会经常遇到apply和bind这两个函数，是时候回顾下这几个函数的意思了

#####1. 首先说下call和apply，这两个函数的作用是设置函数体内this对象的值，显然调用方就是一个函数本身，这两个函数的区别不大，第一个参数是作用域值用来给函数本身的this赋值，或者说改变函数的调用方，看到这个函数的解读应该是这样的：用后面第一个参数作为调用对象然后.上这个函数， apply的第二个参数可以是Array实例，也可以是arguments对象，而call函数必须将传入的参数一个个列出来，看一段代码

这里说明了apply和call函数的用途，他们将对象和方法解耦了，方法不必依赖对象存在，一个方法可以给多个对象用，这个相对于java确实有很大不同，java的对象和方法是完全绑定的，函数在执行时的this在编译时就已经确定好了，而javascript则是要在运行时确定

#####2. apply由于第二个参数是数组或arguments对象，又多了一个相应的用途，即某些函数接收多个参数作为输入，但是一个个写参数比较麻烦或者有的参数本来就已经是个数组形式了，比如通过api获取的数据，那么用apply就比较方便，看下面一段代码

mysql的中文处理

发表于 2017-02-08   |   分类于 mysql

mysql默认的字符编码可以通过show variables like ‘character%’ 来查看，

client 为客户端使用的字符集。
connection 为连接数据库的字符集设置类型，如果程序没有指明连接数据库使用的字符集类型则按照服务器端默认的字符集设置。
database 为数据库服务器中某个库使用的字符集设定，如果建库时没有指明，将使用服务器安装时指定的字符集设置。
results 为数据库给客户端返回时使用的字符集设定，如果没有指明，使用服务器默认的字符集。
server 为服务器安装时指定的默认字符集设定。
system 为数据库系统使用的字符集设定。

  1. sudo cp /usr/local/mysql/support-files/my-defalut.cnf /etc/my.cnf

  2. 停掉mysql ， sudo vim /etc/my.cnf ，加入

    [mysqld]

    character-set-server=utf8
    collation-server=utf8_general_ci

    [client]

    default-character-set=utf8

  3. ALTER DATABASE database_name CHARACTER SET utf8 COLLATE utf8_general_ci

（或通过sequel在database中）

  1. ALTER TABLE table_name CONVERT TO CHARACTER SET utf8 COLLATE utf8_general_ci(或通过sequel在table info中)
