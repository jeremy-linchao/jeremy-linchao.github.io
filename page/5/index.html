git忽略已经被提交的文件

发表于 2017-04-21   |   分类于 git

现在项目的根目录放了 .gitignore 文件，并且git远程仓库的项目根目录已经有了 logs文件夹, 但是之后想忽略日志里的内容，光改写.gitignore并不够，正确做法是：

  1. git rm –cached logs/xx.log
  2. 然后更新 .gitignore 忽略掉目标文件
  3. 最后git commit -m “We really don’t want Git to track this anymore!”

git冲突解决

发表于 2017-04-21   |   分类于 git

  1. 每次git合并时遇到冲突总会感觉非常崩溃，其实对于冲突一点都不需要过于担心，因为总是可以通过git merge - - abort来撤销合并恢复到之前的状态

  2. 当两个改动发生在同一个文件的同一些行上，我们就要看看发生冲突的文件的内容了。Git 会非常友好地把文件中那些有问题的区域在 “<<<<<<< HEAD” 和 “>>>>>>> [other/branch/name]” 之间标记出来。

    中间用=======来作分割， 第一个HEAD到======表示本地版本的改动， ======到最后那个分支名表示那个版本，理解了git的这个标记的意思非常重要。
