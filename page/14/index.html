Angular 11.22

When a Controller is attached to the DOM via the ng-controller directive, Angular will instantiate a new Controller object, using the specified Controller’s constructor function. A new child scope will be created and made available as an injectable parameter to the Controller’s constructor function as $scope.

这段话翻译过来的意思是说当ng-controller被加到一个dom元素中会发生什么？首先Angular会来初始化一个controller对象，用的就是controller这个构造函数，然后会有一个新的子scope会被创建出来，被当成一个参数传到构造函数里去

Use controllers to:

  • Set up the initial state of the $scope object.

  • Add behavior to the $scope object.

    使用controller主要就来做两件事，第一件是初始化$scope这个变量，另一件事是给$scope加上行为

Vim 环境打造

发表于 2016-11-16   |   分类于 utility

1. 思路是使用spf13来配置vim，https://github.com/spf13/spf13-vim，可以节省比较多的时间

2. 几个常用插件使用方式

  • 注释和取消注释： ，+ c + space

  • 查找文件： ctrl + p ，然后使用 ctrl + j,k 来上下移动

  • 打开工程目录： ctrl + e, 然后使用t来打开文件，在左边和右边窗口移动使用ctrl + H，L来左右移动鼠标，在多个tab之间使用gt来切换

  • 颜色配置：在.vimrc那个文件找到solarized那一段，let g:solarized _termcolors = 16 这样的话文本不会太暗

  • 字体配置： 首先把let g:airline_powerline_fonts=1这段写到.vimrc.before.local，同时在终端的perference上配置下字体为SourceCodePro

3.暂时不用的插件先都关闭掉

  • echo UnBundle \’AutoClose\’ >> ~/.vimrc.bundles.local 把不需要的插件写到.vimrc.local，然后:BundleClean! 移除插件目录

bower

bower是一个前端包管理器，用来管理框架、库、assets和utilities
  • bower会记录所有的包并且保证他们是最新的
  • bower通过bower.json这个文件来管理所有的包
  • bower安装通过npm install -g bower
  • 创建bower.json bower init
  • 保存依赖 bower install PACKAGE –save

    webpack

    webpack是一个模块bundler，它会生成静态的asserts，

现在的网站都开始转向为web app，因此会有以下几个趋势

  • 越来越多的javascript会被用到
  • 浏览器提供的接口更多
  • 很少有整个页面的重新加载

所以这就导致了大量的客户端代码，因此需要一个模块化打包系统来帮助把你的代码库改造成模块

webpack 是为了解决什么问题？答案是：文件依赖管理。浏览器端的 Js, 出于安全的考虑，对本地文件没有操作权限，不能引用其它 js, css 等文件。而 webpack 就是用来解决这个问题的，让你的项目可以很好地分文件、分模块，而且它对外部文件的引入同时支持 cmd, amd 和 commondJs 这三种形式，够有诚意。

或许你要说了，解决文件依赖，早在 require.js 和 sea.js 的时候，都已经解决了呀！那么，webpack 在这方面，有哪些新的突破：

  • 支持引用各种拓展名的文件
  • 能够在不依赖 gulp 或 grunt 的情况下直接产出打包文件
  • 支持实时编译，浏览器同步刷新

Angular学习笔记

第0节 Bootstraping

几个核心概念

  • ng-app 标记HTML使得angular可以来找到根元素
  • angular.js 这个js脚本注册了个回调函数，当HTML页面加载好之后，然后就去找ngAPP指令，找到之后就开始bootstarp
  • 绑定 使用 两个花括号 之所以叫绑定是因为如果变量的内容改变了的话，页面显示也会实时改变，不需要手动再发ajax更新了
  • angular-seed 程序骨架

页面加载发生了什么：页面会加载angular.js这个脚本，这个脚本会注册一个回调函数，这个回调函数在浏览器加载完页面之后会来查ngApp这个指令所在的位置，并从相应位置开始渲染

  • Instead of creating a static HTML page on the server, the client-side code “takes over” and interacts dynamically with the view, updating it instantly to reflect changes in model data or state, usually as a result of user interaction

  • The template (the part of the view containing the bindings and presentation logic) acts as a blueprint for how our data should be organized and presented to the user. The controller provides the context in which the bindings are evaluated and applies behavior and logic to our template.
