go基础

发表于 2018-06-21   |   分类于 go

  1. go不像别的语言有多个workspace（每个workspace单独git管理），而是只设一个workspace，这个workspace下会有src，pkg，bin三个目录，src下面会存放项目各自的repo，pkg存放的是包被编译后的文件，为的是可以节省编译时间，bin放的是可执行文件，项目下编译出来的二进制文件都会放到bin下

  2. 几个重要的go tool命令及解释:

    go build: 编译包及里面的相关依赖，但不会去安装到bin或pkg下

    go install: go build 加上安装到bin或pkg

    go get: 下载包及相关依赖，然后安装这些包

    这里有一个问题： 如果go get安装了某个包依赖a，然后这个包a又依赖了b，go get能下载到b的源码吗？

  3. channel

    首先要明白发和收的写法，<-c，c如果是channel，这个是收，c<- 0， 这个是发送，再然后就是对于buffered和unbuffered channel的区别

    如果是buffered，发送在接收之前； 如果是unbuffered，接收在发送之前

A send on a buffered channel happens before the corresponding receive from that channel completes.

The kth receive on a channel with capacity C happens before the k+Cth send from that channel completes.

A receive from an unbuffered channel happens before the send on that channel completes.
