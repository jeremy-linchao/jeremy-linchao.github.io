你不知道的javascript

发表于 2017-08-31   |   分类于 javascript

  1. javascript标准总共只有7个类型，null, undefined, boolean, number, string, object, symbol, 至于常用的array和function其实都是object下的子类型。这7个类型用typeof并不能一一对应地打印出相应字符串，typeof null的结果是object，这是一个js的bug，但不太可能去修复这个bug，以为太多的网站依赖这个bug了

  2. javascript的变量是没有类型的，真正有类型的是值，具体来说，JS的变量可以在一个赋值语句中被赋值为number,可以在接下来一个赋值语句中赋值成string。 因此如果你在JS用typeof， 它的意思是在问这个变量的值是什么类型，而不是变量本身

  3. undefined并不是undeclared

    对一个未声明过的变量typeof， 返回的是undefined

  4. JS的数据不要求里面的元素都是一个类型，因此声明成下面这个也不要感到奇怪

    JS的数组是可以用字符串来做索引的，原因在于数组其实也是个object， 但是用字符串索引的是不会被记到length里去的

    一般来说也建议不要用数组的字符串索引

  5. 将array-like的元素，比如dom元素转化为真正的数组(这样就可以调用诸如indexOf, forEach等方法了) ，有两种实现方式，一种是用Array.prototype.slice.call(arguments)

    另一种是ES6的Array.from

  6. JS的string和字符数组一样吗？表面看上去他们有点一样，比如都有indexOf方法，都有concat方法，都有length属性，但实际上他们很不一样。一个最大的区别在于说string是不可以改变的，

    另外string本身也没有join，reverse方法，因此一个常见的面试提： 给你一个字符串，让你去反转，这个该如何操作呢？可以先把字符串转化成字符数组，然后调用相应方法，最后再join

    注： 书上说对于multibype characters无效，我测试中文是可以的

    This approach doesn’t work for strings with complex (unicode) characters in them (astral symbols, multibyte characters, etc.).

  7. JS的number类型是直接可以调用Number的方法的，比如保留小数点位数的toFixed()方法

  9. JS的整形最大值是2的53次方-1，这个数还是要记住的，因为在某些情况下会有用，比如定义数据库的ID时，不能用64位的整数，只能用string来存，使用Number.MAX_SAFE_INTEGER这个来定义，最小值是-9007199254740991，使用Number.MIN_SAFE_INTEGER这个来定义，

    浮点型的最大值是1.798e+308，使用Number.MAX_VALUE这个来定义，最小值是5e-324，使用Number.MIN_VALUE这个来定义

  10. 32为有符号整数： 为什么需要32位整数呢？ 有些操作只能用于32位整数，比如位操作。把一个number类型的数变为32位有符号数的方式是 | 0

  11. void 操作符，确保某个表达式的返回值是undefined，他不会修改已有的值

  15. natives，比如String(), Number(), 这些有点像其他语言的构造函数， 为什么要这些natives，因为基本类型没有属性方法，比如一个string，没有.length属性，没有toUpperCase()方法， number没有.toFixed()方法， JS默认会自动会基本类型加上包装

    比如你顶一个string，是可以直接调用toUpperCase()方法

  17. ​ 经常看到console.log打印对象会打出诸如”[object Object]”这样的东西，这是因为如果没有实现对象的toString()方法，默认打出的是[[class]]属性

  18. JSON.stringify()会自动忽略掉 undefined, null 和 symbol，如果这些值是在数组中，会用null来代替，如果是在一个对象中，则这个属性会被排除掉

  19. JSON.stringify是有两个可选参数的，一个叫replacer，可以是数组或是函数，用来指定对象中哪些需要保留在最终的json字符串中，哪些需要剔除，而第三个参数是为了美观用的，指明每一行的分隔符

    var a = {

    b: 42,
c: "42",
d: [1,2,3]

    };

    JSON.stringify( a, null, 3 );
    // “{
    // “b”: 42,
    // “c”: “42”,
    // “d”: [
    // 1,
    // 2,
    // 3
    // ]
    // }”

    JSON.stringify( a, null, “—–” );
    // “{
    // —–”b”: 42,
    // —–”c”: “42”,
    // —–”d”: [
    // ———-1,
    // ———-2,
    // ———-3
    // —–]
    // }”

    var a = 42;
    var b = String( a );

    var c = “3.14”;
    var d = Number( c );

    b; // “42”
    d; // 3.14

    var d = new Date();

    +d; // 1408369986000

    ~42; // -(42+1) ==> -43

    var a = “Hello World”;

    ~a.indexOf( “lo” ); // -4 <– truthy!

    if (~a.indexOf( “lo” )) { // true
    // found it!
    }

    ~a.indexOf( “ol” ); // 0 <– falsy!
    !~a.indexOf( “ol” ); // true

    if (!~a.indexOf( “ol” )) { // true
    // not found!
    }
    ```

    ~~有点像floor取整，但他们不一样！

    ~~的原理是先转换成一个Int32，然后取反，第二个~再取反，所以小数部分会被结掉，但这个有试用范围，首先只适用于32位的整数，其次只适用于正数
