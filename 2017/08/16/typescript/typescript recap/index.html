typescript recap

发表于 2017-08-16   |   分类于 typescript

Typescript用了也有一段时间了，是时候从头到尾仔仔细细过一遍文档了，决定还是看英文原版的，虽然可能会稍微累一点

1.基础类型:

  1. 增加了枚举类型： 个人觉得枚举是一个非常重要的数据类型，工作中经常用到，特别好的一点是从typescript2.4开始支持字符串枚举了，即枚举值是字符串，以前的版本枚举值只能是数值，在很多场景中都有应用，一个例子就是redux的action，之前是用常量来定义，但如果用枚举值可以更方便地管理； API的返回值很多可以用枚举值了

    另外需要注意的是对于枚举值是数值的枚举类型，是可以从值来返回枚举名字本身的

  2. 类型断言：当我们比编译器知道更多关于一个变量的类型的时候可以用类型断言，有点像其他语言里的强制类型转换，但它实际上不会对数据做任何操作，仅供编译器用，有两种写法，一种是<>

    另一种是as

2.声明

  1. var是函数作用域， let是块级作用域，还是再看下这个经典的例子

3. 接口

  1. 对象的readonly属性，即这个属性只在对象初始化时改变，之后就不能再被修改了，就像使用const一样尽可能使用readonly

  2. ReadonlyArray, 不可变数组

  3. 对象字面量在赋值给其他变量或函数参数时会被加强检查的，说的是如果对象字面量里有任何属性目标对象里没有时就会报错，这里的函数参数是用对象字面量，但color拼错了，在js这没有问题，但ts这里会报错

  5. TS对函数做了增强，表现在TS中的函数每一个参数都是必须要传的，除非显示指定为可选参数或者默认参数，这和JS有很大不同，JS中所有参数参数都是可选的，如果你不传，表示你传的是undefined

  6. 默认参数是ES6就有的

  7. 剩余参数也是ES6有的，即多余的函数参数不在放到arguments里，而是用…一个剩余参数

  8. 箭头函数显示绑定this, Arrow functions capture the this where the function is created rather than where it is invoked

  9. TS支持函数重载：对同一个函数作多个声明
