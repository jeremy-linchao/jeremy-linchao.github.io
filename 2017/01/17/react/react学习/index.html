react学习

发表于 2017-01-17   |   分类于 react

正如研究任何一种框架一样，研究react也要搞清楚作者开发他的动机，其实说白了就是前段组件化，这样的好处是可以复用很多代码，不仅仅是自己写的组件，别人的组件也可以，这是不是很酷呢？

既然是组件，那组件势必要把view显示和逻辑封装在一起，这个就和很多之前的前端MVC开发方式有很多不同，是需要适应的，但我觉得组件这个东西并不新颖，在UI开发的历史上早就存在

既然是组件，有几个概念很重要，compenent不用多说，那如何把数据传到component中呢，比如我有一个表格需要显示数据，怎么把数据传进来呢？组件是怎么来处理css的呢，毕竟组件需要提供很多不同显示样式？这些都是需要带着的问题去看react的

  1. prop和state的区别，他们是用来干嘛的? 这里有句话：State is similar to props, but it is private and fully controlled by the component. 显然他的意思是说state是组件自有的属性，完全私有。那么它用来做什么呢？举个例子，有个时钟的组件，时钟每秒钟都得更新下，如果用props是不可以的，因为props传进来是不能改变的，而state是可变的，同时对于时钟来说，时间这个属性本来就是完全它自己的，不需要外面传数据进来。

  2. react的数据是单向绑定，state只能用setState函数来修改(除了构造函数)，state的更新可能是异步的

  3. react的事件命名是驼峰的，事件处理器本身是一个JSX表达式

    https://www.smashingmagazine.com/2014/01/understanding-javascript-function-prototype-bind/

    js类里的成员函数默认是不绑定的，这篇文章解释的比较清楚

  4. react-router如何与ant-design的组件结合?

    有一个问题一直想不通，就是像ant-design的menu组件不是都有超链接了嘛，router这个link又是用来干嘛呢？这里一定要搞清楚一个问题，在menu组件里的连接可以认为就是带了标签的跳转，但是跳转到哪儿去是有react-router里的各条route来决定的，他们之间并不冲突

  5. Components must return a single root element，所以return的element必须加一个包裹

  6. All React components must act like pure functions with respect to their props

  7. 当把函数式组件改为class组件时，有两点需要注意，1）多加了个render函数，里面定义个return函数，这其实是个闭包 2）不需要写函数参数props，反而是在函数需要用到props的时候写this.props.date

  8. While this.props is set up by React itself and this.state has a special meaning, you are free to add additional fields to the class manually if you need to store something that is not used for the visual output.

    If you don’t use something in render(), it shouldn’t be in the state.

  9. 关于setState的三个注意点： 1）所有修改state数据只能用this.setState方法，换句话说除了在构造函数能访通过this.state=的形式来设置state，其余地方都只能用函数，但是访问是没有问题的 2）states的更新可能是异步的，React may batch multiple setState() calls into a single update for performance. 要更新的话需要用setState的第二种用法，参数传一个回调函数 3）states的更新是合并在一起的，但不同的setState不会影响各自

  10. 事件处理一般是类的成员函数，但是需要在构造函数里绑定this或者成员函数写成=的形式

  11. 渲染多个组件可以使用map()函数

  12. Keys should be given to the elements inside the array to give the elements a stable identity:

  13. 受控组件： 比如一个最基本的文本输入框，正常文本框的显示是html自身组件控制的，但是如果你加上了state状态，这个文本值的显示是完全受控的

  14. lifting state的例子还没有完全搞懂

  15. composition是什么意思？这点没搞明白
