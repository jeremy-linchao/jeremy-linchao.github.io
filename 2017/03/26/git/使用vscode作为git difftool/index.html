使用vscode作为git difftool

发表于 2017-03-26   |   分类于 git

当学习大牛的代码时，git往往是一个非常重要的工具，原因在于git可以看到从最开始的版本到最新的版本，了解大牛是怎么一步步写代码的，所以最好的打开方式是git checkout到最初的一个版本，然后一个一个版本看，这个方法之前有讲过，但是这里有个问题，如何直观的看到不同版本的变化呢？github当然是可以，但github毕竟需要网络，在很多没有网络的情况下就无法使用了，另一个就是git 命令行自带的git diff工具，但命令行自带的也不便于观看，所以想到了是否可以用vscode呢？经过一番搜索，总结一下

  1. 确保vscode可以在命令行下打开:

    shift + cmd + P，然后输入shell comand, 找到install ‘code’ command in PATH

*2. 将vscode作为git默认的editor(这步可以不用，默认的editor还是用vim比较好，否则在git merge的时候会有些问题)

  1. 配置git difftool

  2. 比较

    如果只比较当前版本和之前一个版本，可以更简单
