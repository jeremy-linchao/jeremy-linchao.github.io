typescript学习

发表于 2017-03-23   |   分类于 typescript

typescript是javascript的超集，那超在哪儿？

1. 基本类型增强
  1. tuple： 长度固定的数组，但里面的值不需要是同一个类型

如果访问超过了tuple的索引范围，就会使用union type

  1. 枚举类型： 所谓枚举无非是给数值类型起一个比较友好的名字罢了，提高开发效率，比如通信状态的返回值可能有很多，用枚举远比数字类型来的直观，没有枚举的话就需要使用对象了
  1. any类型: 当不确定某个值的类型时可以用any来通过编译
  2. void类型：没有类型，多用于函数返回值
  1. never类型：永远不会发生的类型，什么情况下会有用？ 如果一个函数就是抛一个错，那么它的返回值就可以是never
  1. 类型断言：当我们确定某个值是什么类型的时候，像是其他语言里的类型转换，但是不会做类型检查和转换，也不会有运行时的影响，只是单独给编译器使用
2. 类型声明

  1. var的作用域会泄露到包含它的函数或者包含它的模块

  2. 在同一个函数中var声明的变量可以重复声明多次

  3. 在循环里声明的let和普通的let有很大区别，循环里的let在每次递归时都会创建一个独立的作用域， 再看下这两段经典的代码：

3. interface
  1. interface是ts的一个核心，它主要用来检查变量值的类型，因为javascript的变量的类型是可变的，实际上是它的值的类型是可变的，interface起到这个作用，它来定义某个变量可以有的类型，因此在编译阶段就能发现此类错误
4. 可索引的类型

​ 可以用字符串或者数字来作索引，但是数字索引的返回值必须是字符串索引返回值类型的子类型。 这是因为当使用number来索引时，JavaScript会将它转换成string然后再去索引对象。
