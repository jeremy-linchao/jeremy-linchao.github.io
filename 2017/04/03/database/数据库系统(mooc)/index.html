数据库系统(mooc)

发表于 2017-04-03   |   分类于 database

  1. 关系与表的关系： 关系是关系模型里一个数学概念，而表是DBMS实现时用的一个称呼，两者在大多数情况下都是相同的，但关系里不能有重复的record，表里可以有

  2. 如何描述一个关系: 1) 先确定域，域的所有取值是能够确定下来的，同时有多少个域也是能确定下来的

    2）使用域作笛卡尔乘积，从中挑选一些有意义的record的集合作为一个关系

  3. 完整性约束： 1） 实体完整性： 主键不能为空 2）参照完整性： 外键的值的取值为null或者为参照表的主键中的取值 3） 自定义完整性： 用户自己定义的约束，比如字符串长度，数字长度

  4. 候选键（能唯一标识所有用户的属性组） 主键： 从候选键中选出一个属性组 外键： 在某一个关系下不是候选键，但在另一个关系下是主键，用来连接关系​

  5. 关系代数的基本动作： 并， 差， 乘积， 选择， 投影

  6. 并相容性： 两个关系的属性的数量以及每个属性的取值范围必须相同。 并，交， 差运算需满足并相容性

  7. theta-join, 等值连接: 特殊的theta-join

  8. Natual-join: 特殊的等值连接

  9. 外连接： 为什么需要外链接？内连接由于是需要属性相同并且值也相同，因此会丢失一部分信息。

    定义：连接时如果在另外一张表中找不到对应的元祖，则以空值作为代替连接

  10. DDL: CREATE, ALTER, DROP

    DML: INSERT, DELETE, UPDATE, SELECT

    DCL: GRANT, REVOKE

  11. 批量插入 INSERT INTO Table(col1, col2) 子查询

  12. 删除： DELETE FROM 表名 [WHERE 表达式]

  13. 更新： UPDATE 表名 SET 列名 = 表达式|（子查询）[WHERE 条件表达式]

  14. 子查询： 出现在Where子句中的Select语句， IN 与 NOT IN； theta some | all

  15. E-R模型： Enitity-Relation Model，四个要素：实体、属性、联系、关键字

  16. 一对多和多对多关系的区别: 一对多的联系可以通过外键来保存，对多对的联系只能单独再建一个联系表

  17. 一对多联系如何转换： 将单方参与实体的关键字作为多方参与实体对应关系的属性
